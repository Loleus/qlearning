Legenda zmian parametrów

real-time od nast. epizodu reset epizodów reset Q

Real-time oznacza, że parametr wpływa od razu na kolejną decyzję lub aktualizację. Od następnego epizodu oznacza, że bieżący epizod dobiega do końca według starych ustawień. Reset epizodów zeruje licznik epizodów i historię, ale zachowuje wiedzę. Reset Q czyści tablicę Q i również resetuje epizody.

Strategie eksploracji

1. ε-greedy klasyczny

a = { losowa akcja z prawd. ε, argmaxₐ Q(s,a) w przeciwnym razie }

Podstawowa strategia ε-greedy bez dynamicznego zmniejszania ε. Stała wartość eksploracji.

  • ε — real-time

1.1. ε-greedy z dynamicznym ε

a = { losowa akcja z prawd. ε, argmaxₐ Q(s,a) w przeciwnym razie }

Dodany został harmonogram eksploracji z minimalnym ε i wzmocnieniem przy stagnacji. Dzięki temu polityka może znów zwiększyć eksplorację, gdy skuteczność epizodów spada.

Dostępne typy decay: Linear (ε = ε₀ - k·t), Exponential (ε = ε₀ · decayᵗ), Inverse (ε = ε₀ / (1 + k·t)), Cosine annealing (ε = ε_min + ½(ε₀-ε_min)(1+cos(πt/T))).

  • Typ decay ε — od nast. ep.
  • ε start — real-time
  • ε min, decay, boost — od nast. ep.

2. Softmax / Boltzmann

P(a|s) = exp(Q(s,a)/τ) / Σᵢ exp(Q(s,i)/τ)

Im wyższa temperatura τ, tym bardziej "miękki" i eksploracyjny wybór. Gdy τ maleje, polityka coraz częściej wybiera najlepsze akcje.

  • τ start — real-time
  • τ min, decay — od nast. ep.

3. Optimistic Q-initialization

Q₀(s,a) = stała dodatnia > 0

Nowe stany otrzymują zawyżone początkowe wartości, więc agent początkowo "wierzy", że nieodwiedzone akcje są opłacalne. To działa jak eksploracja bez dużego ε. Ta strategia jest dostępna tylko dla tablicowego Q-learningu.

  • Q₀ — reset Q + epizodów
  • Zmiana strategii na/z optimistic — reset Q + epizodów

4. UCB

UCB(s,a) = Q(s,a) + c · √( ln(N(s)+1) / (N(s,a)+1) )

Składnik ufności premiuje rzadko wybierane akcje. Gdy licznik odwiedzin rośnie, bonus eksploracyjny maleje i polityka przechodzi do eksploatacji.

  • c — real-time
  • Zmiana strategii na UCB — reset epizodów

4.1. Hybrydowa (UCB + ε-greedy)

UCB(s,a) = Q(s,a) + c · √( ln(N(s)+1) / (N(s,a)+1) ) a = { losowa akcja z prawd. ε, argmaxₐ UCB(s,a) w przeciwnym razie }

Kombinacja UCB z ε-greedy. Z prawdopodobieństwem ε wybiera losowo, w przeciwnym razie używa UCB do wyboru akcji.

  • c UCB — reset Q
  • ε — real-time

5. NoisyNet-style

Q̃(s,a) = Q(s,a) + σ · ξ, gdzie ξ ~ N(0,1)

Zastosowanie eksploracji typu NoisyNet przez szum Gaussowski dodawany do wartości akcji przy wyborze ruchu. Pozwala to badać przestrzeń bez ręcznie ustawianego ε.

  • σ start — real-time
  • σ min, decay — od nast. ep.

Double Q-Learning

Idea Double Q-Learning

Losowo: Qᴬ ← Qᴬ + αδᴬ  lub  Qᴮ ← Qᴮ + αδᴮ δᴬ = r + γQᴮ(s', argmaxₐ Qᴬ(s',a)) − Qᴬ(s,a) δᴮ = r + γQᴬ(s', argmaxₐ Qᴮ(s',a)) − Qᴮ(s,a)

Double Q-Learning (Hasselt, 2010) rozwiązuje problem przeuczenia (overestimation) w standardowym Q-learningu. W standardowym podejściu ta sama funkcja Q wybiera i ocenia akcję, co prowadzi do systematycznego zawyżania szacowań wartości szczególnie w stanie początkowym treningu.

Rozwiązanie: utrzymuj dwa niezależne estymatory Qᴬ i Qᴮ. Przy każdej aktualizacji losowo wybierz jeden.

  • Wybór akcji w polityce: średnia arytmetyczna Qᴬ i Qᴮ
  • Aktualizacja Qᴬ: wybierz akcję przez argmax z Qᴬ, ale oceń jej wartość przez Qᴮ
  • Aktualizacja Qᴮ: wybierz akcję przez argmax z Qᴮ, ale oceń jej wartość przez Qᴬ
  • To decoupling zapewnia umiarkowane, nieobciążone szacowanie (Hasselt, 2010)
  • Włączenie Double Q — reset Q + epizodów

Eligibility traces

Eligibility traces λ

eₜ(s,a) ← γλ eₜ₋₁(s,a), Q ← Q + α · δ · e

Dodaje ścieżki nagród λ do aktualizacji tablicowego Q-learningu. Przyspieszają propagację błędu TD do wcześniejszych stanów. W tej implementacji są to ślady zastępujące (replacing traces) z czyszczeniem przy przejściu niezgodnym z polityką zachłanną.

  • Włączenie traces — od nast. ep.
  • Lambda λ — od nast. ep.

Wpływ pozostałych parametrów

  • Rozmiar komórki — reset Q, bo zmienia dyskretyzację stanu.
  • Alpha α, Gamma γ — działają natychmiast.
  • Lambda λ — od następnego epizodu.
  • Prędkość — natychmiast na ruch agentów.
  • Reset epizodów — czyści historię, zachowuje wiedzę Q.
  • Reset Q — czyści wiedzę i resetuje epizody.

Co to jest Q-learning?

Q‑learning to algorytm uczenia ze wzmocnieniem (Reinforcement Learning, RL), który uczy się optymalnej polityki działania w środowisku metodą prób i błędów, bez znajomości modelu środowiska. Kluczowym elementem jest funkcja Q(s, a), która ocenia „jakość” wykonania akcji a w stanie s. Algorytm aktualizuje tę funkcję na podstawie nagród i przewidywanej wartości przyszłych stanów.

Historyczne tło powstania Q‑learningu

1.1. Korzenie: Markowowskie procesy decyzyjne (MDP)

Podstawy teoretyczne sięgają lat 50. i 60., kiedy powstała teoria Markov Decision Processes (Bellman, Howard). Bellman wprowadził słynne równanie:

V(s) = maxₐ [R(s,a) + γ ∑_{s′} P(s′|s,a) V(s′)]

To fundament dynamicznego programowania.

1.2. Problem: dynamiczne programowanie wymaga znajomości modelu

W praktyce często nie znamy:

  • prawdopodobieństw przejść P(s′|s,a),
  • funkcji nagrody R(s,a).

Potrzebny był algorytm, który uczy się optymalnej polityki bez modelu.

1.3. Przełom: Watkins (1989)

Christopher Watkins w doktoracie z 1989 r. zaproponował Q‑learning – algorytm, który:

  • nie wymaga modelu środowiska,
  • uczy się optymalnej polityki off‑policy,
  • gwarantuje zbieżność do optymalnej funkcji Q przy pewnych warunkach.

To był przełom, bo RL stało się praktyczne.

Na czym dokładnie polega Q‑learning

2.1. Funkcja Q

Q‑learning uczy funkcji:

Q(s,a) = oczekiwana suma zdyskontowanych nagród po wykonaniu akcji a w stanie s.

2.2. Reguła aktualizacji

Kluczowy wzór Watkinsa:

Q(s,a) ← Q(s,a) + α [r + γ max_{a′} Q(s′,a′) − Q(s,a)]

Interpretacja:

  • r – nagroda za przejście,
  • γ – współczynnik dyskontowania przyszłych nagród,
  • α – tempo uczenia,
  • max Q(s', a') – najlepsza przewidywana wartość w kolejnym stanie.

2.3. Dlaczego to działa

Aktualizacja jest próbą przybliżenia równania Bellmana:

Q* = T Q*

gdzie operator Bellmana jest kontrakcją → iteracyjne stosowanie prowadzi do zbieżności.

2.4. Charakter algorytmu

  • off‑policy – uczy się optymalnej polityki niezależnie od polityki eksploracji (np. ε‑greedy),
  • model‑free – nie potrzebuje znajomości dynamiki środowiska,
  • tablicowy – klasyczna wersja przechowuje Q w tabeli.

Przykład działania krok po kroku

  1. Agent obserwuje stan s.
  2. Wybiera akcję a (np. ε‑greedy).
  3. Otrzymuje nagrodę r i przechodzi do stanu s′.
  4. Aktualizuje Q(s,a) według wzoru.
  5. Powtarza proces.

Po wielu epizodach Q(s,a) zbiega do wartości optymalnych.
